# Memory Manager — 结构化长期记忆管理

让 Agent 拥有可靠的长期记忆：自动捕获、分层存储、定期瘦身、精准检索。

## 核心架构

```
MEMORY.md (≤5KB)          ← 核心记忆，每次 session 自动加载
├── 🔒 用户偏好（永久）    ← 不删
├── 💼 角色定位（永久）    ← 不删
├── 📝 业务经验（≤15条）   ← 长期保留，满时合并
├── 📋 活跃任务（完成删）   ← 临时
├── 🔄 纠正记录（≤10条）   ← 晋升后删
└── 💬 对话摘要（7天精简）  ← 临时

memory/
├── YYYY-MM-DD.md          ← 每日日志（自动生成）
└── archive/               ← 归档（瘦身时迁入）

.learnings/
└── corrections.md         ← 纠正详细记录（晋升源）
```

## 1. 记忆写入规则

### 自动写入（不需要用户说"记住"）

| 触发条件 | 写入位置 | 示例 |
|---------|---------|------|
| 用户纠正了你 | 业务经验 + .learnings/ | "不对，应该用 CTE" |
| ≥3轮才搞清一个概念 | 业务经验 | 反复讨论后终于理解的概念 |
| 用户做了决策/选方案 | 对话摘要 | "决定用方案 B" |
| 完成复杂任务（≥5步） | 对话摘要 | 怎么做的、关键步骤 |
| 发现数据口径/技巧 | 业务经验 | "这个表的 from 字段要反引号" |
| 用户表达偏好 | 用户偏好 | "我喜欢简洁的报告" |
| 任务被搁置/等待 | 活跃任务 | "等 API key" |

### 手动写入

用户说"记住 XXX"时，按决策树分类：
1. "我喜欢/我的习惯" → `MEMORY.md` 用户偏好
2. "大家都应该知道" → 共享知识（如有）
3. 拿不准 → 先写个人记忆，问用户是否共享

### 打标签（提高检索命中率）

```markdown
[用户: bobo] [话题: SEO] 决定采用方案 B
[话题: 数据库] from 字段需要反引号
```

## 2. 纠正检测与记录

### 检测信号（不限于此）

- 直接否定："不对"、"错了"
- 给出不同答案：你说 A，对方说"是 B"
- 温和引导："其实..."、"应该是..."
- 示范正确做法（不说你错，直接展示）
- 质疑："你确定？"
- 放弃让你做："算了我来" ← 强烈失败信号

### 纠正后流程（同一 turn 内完成）

1. **承认** — 简洁承认，不找借口
2. **理解** — 确保理解正确做法
3. **记录** — 同时写两处：

`.learnings/corrections.md`（详细版）:
```markdown
## YYYY-MM-DD HH:MM — 简短标题
- **纠正**: 正确的做法
- **我的错误**: 我之前做了什么
- **根因**: 为什么犯错
- **教训**: 下次怎么做（一句话）
- **状态**: ⏳待验证
```

`MEMORY.md` 纠正记录区（摘要版，≤10条）:
```markdown
### [YYYY-MM-DD] 简短标题
- 我原来以为: xxx
- 正确理解: xxx
```

### 晋升规则

- 两条相似纠正 → 晋升到 AGENTS.md（永久规则）
- 对他人也有价值 → 写入共享知识

## 3. 记忆检索

### 新 Session 启动时（自动执行）

1. 读取 `MEMORY.md`
2. 至少 2 次 `memory_search`：
   - 搜索用户相关记忆
   - 搜索当前话题关键词
3. 搜不到 → 诚实说"没找到相关记忆"，不编造

### 对话中按需检索

回答关于历史工作、决策、日期、偏好的问题前，先 `memory_search` 再回答。

## 4. 自动瘦身

### 触发条件

- `MEMORY.md` > 5KB
- Heartbeat 检查时发现超限

### 瘦身规则

| 区域 | 策略 |
|------|------|
| 🔒 用户偏好 | 永不删 |
| 💼 角色定位 | 永不删 |
| 📝 业务经验 | >15条时合并相似项 |
| 📋 活跃任务 | 已完成 → 立即删 |
| 🔄 纠正记录 | 已晋升 → 删除 |
| 💬 对话摘要 | >7天 → 精简合并或归档 |

### 归档流程

被删除的内容移入 `memory/archive/` 对应文件，不是真正丢失。

## 5. 每日日志

### 自动生成

路径: `memory/YYYY-MM-DD.md`

### 写入时机

- Compaction flush 前
- 每天最后一次对话结束时
- 重要事件发生时

### 内容

记录当天关键事件、决策、任务进展。已有内容只追加不覆盖。

## 6. 隐私隔离

### 群聊 vs 私聊

- 群聊汇总 *不包含* 私聊内容
- 私聊记忆 *不在* 群聊中引用
- 报告/总结只基于当前会话空间的内容
- 唯一例外：用户在群聊中明确要求分享

## 初始化模板

新 Agent 首次使用时，创建以下结构：

```bash
# 创建 MEMORY.md
cat > MEMORY.md << 'EOF'
# MEMORY.md - Long-Term Memory

## 🔒 用户偏好（永久保留）

## 💼 角色定位（永久保留）

## 📝 业务经验（长期保留）

## 📋 活跃任务（完成后删）

## 🔄 纠正记录（≤10条）

## 💬 对话摘要（7天后精简）
EOF

# 创建目录
mkdir -p memory/archive .learnings
touch .learnings/corrections.md
```
